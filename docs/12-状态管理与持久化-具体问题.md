# 状态管理与持久化系统——具体代码级问题

## 1. useGame.ts — 核心 Hook

### 问题 1.1：`深拷贝` 函数对特殊类型处理不当
**文件**: `hooks/useGame.ts:643-651`
```typescript
const 深拷贝 = <T,>(data: T): T => {
    if (data === undefined || data === null) { return data; }
    if (typeof structuredClone === 'function') { return structuredClone(data); }
    return JSON.parse(JSON.stringify(data)) as T;
};
```
- `structuredClone` 对 `AbortSignal`、`DOM Node` 等类型会抛出异常
- `JSON.parse(JSON.stringify())` 会丢失 `Date`（→字符串）、`NaN`（→null）、`Set`/`Map`（→{}）
**后果**: 存档中时间字段从 Date 变成字符串，NaN 变成 null，导致比较逻辑出错。

---

### 问题 1.2：useEffect 依赖数组不完整导致闭包陈旧
**文件**: `hooks/useGame.ts:630-631`
```typescript
useEffect(() => {
    刷新NPC记忆总结队列(Array.isArray(社交) ? 社交 : [], {
        静默: NPC记忆总结阶段 === 'processing' || NPC记忆总结阶段 === 'review'
    });
}, [社交, memoryConfig]);  // ← 缺少 NPC记忆总结阶段
```
`NPC记忆总结阶段` 是 state 但不在依赖数组中，effect 内部读取的是闭包中的旧值。
**后果**: processing 阶段仍触发 UI 提示。

---

### 问题 1.3：`handleStartMemorySummary` 与 `handleApplyMemorySummary` 竞态条件
**文件**: `hooks/useGame.ts:1378-1389`
```typescript
useEffect(() => {
    if (!待处理记忆总结任务) return;
    if (记忆总结阶段 !== 'remind') return;
    void handleStartMemorySummary();
}, [待处理记忆总结任务, 记忆总结阶段]);

useEffect(() => {
    if (!待处理记忆总结任务) return;
    if (记忆总结阶段 !== 'review') return;
    if (记忆总结错误) return;
    handleApplyMemorySummary();  // ← 自动应用，用户无确认机会
}, [待处理记忆总结任务, 记忆总结阶段, 记忆总结错误, 记忆总结草稿]);
```
`记忆总结阶段` 从 `remind` → `processing` → `review` 快速变化时，两个 effect 可能都执行。`handleApplyMemorySummary` 在 `review` 阶段自动触发，用户没有确认机会。

---

### 问题 1.4：`同步设置社交` ref/state 更新非原子
**文件**: `hooks/useGame.ts:503-509`
```typescript
const 同步设置社交 = (updater: any) => {
    const prev = 社交Ref.current;
    const next = typeof updater === 'function' ? updater(prev) : updater;
    const nextList = Array.isArray(next) ? next : [];
    社交Ref.current = nextList;  // ← 先更新 ref
    设置社交(nextList);          // ← 后更新 state（异步）
};
```
连续调用时，`useEffect` 监听的 `社交` state 可能落后于 ref，导致依赖 `社交` 的 effect 读到旧值。

---

### 问题 1.5：大量未捕获的 Promise 拒绝
**文件**: `hooks/useGame.ts:676, 682, 688, 791, 1692`
```typescript
void dbService.保存设置(设置键.视觉设置, normalized);
void dbService.保存设置(设置键.图片管理设置, normalized);
```

`void` 丢弃 Promise，IndexedDB 写入失败时错误被静默吞掉，无日志无用户提示。
**后果**: 数据丢失无感知。用户以为保存了设置，刷新后丢失。

---

### 问题 1.6：`应用并同步社交列表` 无条件触发自动存档
**文件**: `hooks/useGame.ts:791`

短时间内多次更新社交列表会产生大量并发 IndexedDB 写入。

## 2. storyState.ts — 状态规范化

### 问题 2.1：`深拷贝` 无 null 守卫且不处理循环引用
**文件**: `hooks/useGame/storyState.ts:406`
```typescript
const 深拷贝 = <T,>(value: T): T => JSON.parse(JSON.stringify(value)) as T;
```
- 没有 `null`/`undefined` 守卫（对比 useGame.ts 版本有守卫）
- 循环引用会抛出 `TypeError: Converting circular structure to JSON`
- `Date` → 字符串，`Set`/`Map` → `{}`，`NaN` → `null`
**后果**: 特殊构造的导入数据可能导致整个导入流程崩溃。

---

### 问题 2.2：旧坐标地图字段仍被规范化
**文件**: `hooks/useGame/storyState.ts:1842-1850, 1979-1982`
```typescript
地图: Array.isArray(world?.地图) ? world.地图.map(...) : [],
建筑: Array.isArray(world?.建筑) ? world.建筑.map(...) : [],
地图建筑: Array.isArray(world?.地图建筑) ? world.地图建筑 : [],
地图道路: Array.isArray(world?.地图道路) ? world.地图道路 : [],
地图人物: Array.isArray(world?.地图人物) ? world?.地图人物 : [],
```
AGENTS.md 明确说"不要 reintroduce 旧坐标地图字段"，但 `规范化世界状态` 仍在规范化这些字段。AI 在变量命令中写入旧坐标字段时会被保留。
**后果**: 旧坐标数据残留，地图渲染出现幽灵标记或坐标偏移。

---

### 问题 2.3：`创建开场空白世界` 仍包含废弃坐标字段

```typescript
    地图: [], 建筑: [], 地图层级: [], 地图建筑: [], 地图道路: [], 地图人物: [],
    // ...

```
初始化时就包含废弃字段，规范化函数会继续处理它们。

### 问题 2.4：`补全门派职位` 依赖对象键顺序
**文件**: `hooks/useGame/storyState.ts:278-280`
```typescript
let contributionRank = fallback !== '无' || totalContribution > 0 ? '杂役弟子' : '无';
Object.entries(门派职位贡献门槛).forEach(([rank, required]) => {
    if (totalContribution >= required) contributionRank = rank;
});
```
`Object.entries()` 遍历顺序依赖对象键插入顺序。如果键顺序被意外修改，低贡献阈值可能覆盖高贡献阈值。
**后果**: 高贡献玩家被赋予低职位。

---

### 问题 2.5：`按性别比例取性别` 比例不精确
**文件**: `hooks/useGame/storyState.ts:1388-1392`
```typescript
if (ratio.男 > 0) pool.push(...Array(Math.max(1, Math.round(ratio.男 / total * 20))).fill('男'));
if (ratio.女 > 0) pool.push(...Array(Math.max(1, Math.round(ratio.女 / total * 20))).fill('女'));
```

`Math.round` 四舍五入误差 + `Math.max(1, ...)` 确保每种性别至少 1 个条目。小比例性别的权重被放大，pool 大小不固定。

## 3. dbService.ts — IndexedDB 持久化

### 问题 3.1：`初始化数据库` 未处理 `onblocked` 事件
**文件**: `services/dbService.ts:777-800`
```typescript
export const 初始化数据库 = (): Promise<IDBDatabase> => {
    return new Promise((resolve, reject) => {
        const request = indexedDB.open(DB_NAME, VERSION);
        request.onerror = () => reject(request.error);
        request.onsuccess = () => resolve(request.result);
        // ← 缺少 onblocked 处理
    });
};
```
另一个标签页打开较低版本数据库连接时，`open` 被阻塞，Promise 既不 resolve 也不 reject，永久挂起。

---

## 4. stateHelpers.ts — 状态命令执行引擎

### 问题 4.1：`记忆系统` 命令被静默丢弃 🔴
**文件**: `utils/stateHelpers.ts:333-377`
```typescript
const 根路径列表 = [
    '任务列表', '约定列表', '记忆系统',  // ← 包含记忆系统
    '角色', '环境', '社交', '世界', '战斗', '剧情'
] as const;

const 写回根路径 = (root, next) => {
    switch (root) {
        case '角色': result.char = next; break;
        case '任务列表': result.tasks = next; break;
        // ← 缺少 case '记忆系统': result.memory = next;
        default: break;  // ← 记忆系统走到这里
    }
};
```
**根因**: `命令结果结构` 类型缺少 `记忆系统` 字段，`写回根路径` switch 无对应 case。
**后果**: AI 对记忆系统的命令被解析后静默丢弃，记忆系统永远不会被 AI 更新。

---

### 问题 4.2：`应用路径命令` 数组越界写入产生稀疏数组
**文件**: `utils/stateHelpers.ts:195-215`
```typescript
if (typeof token === 'number') {
    if (!Array.isArray(cursor)) return draft;
    if (cursor[token] === undefined) {
        cursor[token] = typeof nextToken === 'number' ? [] : {};

```
`tokens = [5]` 而 `cursor` 是空数组时，`cursor[5] = {...}` 创建长度为 6 的稀疏数组，索引 0-4 是 `empty`。`JSON.stringify` 把 `empty` 序列化为 `null`。
**后果**: 数组中出现意外的 `null` 元素，导致渲染错误。
