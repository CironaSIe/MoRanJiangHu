# 界面功能与用户体验审计

## 1. 设置项完整性

### ✅ GameSettings.tsx — 所有设置字段均有 UI 开关 + 实时持久化

- 主剧情消息模式、字数要求、字数不足处理、叙事人称、剧情风格
- NTL后宫档位、研发诊断模式、行动选项、防抢话、COT伪装注入
- 免责声明、标签检测完整性、标签自动修复、生成失败自动重试
- 标签协议失败自动回炉、回合结束自动存档、回合提示音
- 繁体模式、全局非流式输出、女主剧情规划
- NSFW模式、男娘/扶她NSFW内容、亲密边界机制
- 饱腹口渴系统、修炼体系、额外提示词
- 游戏初始时间修复

### ✅ ApiSettings.tsx — 接口配置完整

- 多配置管理（创建/删除/切换）
- 供应商选择、Base URL、API Key
- 最大输出 Token（8K/32K/64K/自定义）、温度、Top P
- 主剧情模型选择（获取列表/手动输入/自动推荐）
- DeepSeek/GLM 稳定救场模型配置
- 连接测试功能

### ✅ MapModelSettings.tsx — 地图设置完整

- 地图生成功能总开关
- 回忆解析（带流式输出窗口）
- 全量地图生成独立模型/API配置
- 正文后自动地图更新独立模型开关

### ⚠️ 潜在问题

1. **GameSettings.tsx:84** — `当前主剧情消息模式` 通过 `form.主剧情消息模式 || (form.启用GPT模式 ? 'GPT' : 'Gemini模式')` 计算。两个开关存在语义重叠，新玩家可能困惑
2. **GameSettings.tsx:708-718** — "GPT模式"开关描述说"控制主剧情链路"，但实际主剧情消息模式已通过下拉选择独立控制

## 6. 一致性

### ✅ 优点
- 所有设置面板统一使用 `ToggleSwitch`
- 所有模态框统一金色边框+暗色背景
- 操作确认通过 `requestConfirm` prop 注入

### ⚠️ 不足
1. **确认弹窗不统一** — 部分用 `requestConfirm`，部分用原生 `window.confirm`（`SaveLoadModal.tsx:651`、`GameSettings.tsx:181`）
2. **保存按钮位置不一致**：
   - `GameSettings.tsx:947-951` — 底部右侧
   - `ApiSettings.tsx:473-479` — 顶部 sticky 右侧
   - `MapModelSettings.tsx:349-353` — 底部全宽
3. **删除按钮交互不一致**：
   - `SaveLoadModal.tsx` — hover 显示
   - `AuctionHouseModal` — 始终可见

## 综合评估

| 维度 | 评级 | 说明 |
|------|------|------|
| 设置项完整性 | **优秀** | 几乎所有设置字段都有对应 UI |
| 用户引导 | **良好** | 有提示文字，但缺全局新手引导 |
| 错误反馈 | **良好** | 关键操作有确认，部分用 alert 不够友好 |
| 数据验证 | **良好** | 数值有校验，URL/Key 格式校验缺失 |
| 状态可见性 | **优秀** | 加载/操作状态反馈充分 |
| 一致性 | **良好** | 风格统一，保存按钮位置有差异 |
| 移动端体验 | **优秀** | 安全区域、触摸目标、响应式都有考虑 |
| 性能感知 | **良好** | 大部分有进度，导出全部缺百分比 |

**最高优先级建议**：
1. 统一确认弹窗（全部改用 `requestConfirm`）
2. 增加 API Base URL 格式校验
3. 导出全部存档添加进度百分比
4. 移动端 TopBar 触摸目标加大间距
