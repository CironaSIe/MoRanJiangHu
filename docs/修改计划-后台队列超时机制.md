# 后台队列超时机制完整修复计划

## 设计意图

**核心原则**：
1. 后台队列各阶段应有超时保护，防止 AI 端点挂起导致 UI 永远"处理中"
2. 超时时间应可由用户配置（不同模型速度差异大）
3. 流式场景应检测"无新数据到达"（idle timeout），不只是总时长
4. 用户手动触发单个步骤时不受超时限制
5. 超时后跳过该阶段，不阻塞后续阶段

## 当前问题

```
后台队列流程：
  文章优化 → 变量生成 → 动态世界 → 规划分析 → 地图更新

问题：
1. 变量生成接口不传 signal → 无法取消
2. 地图更新传了 controller.signal 但 controller 不会自动超时
3. 所有超时硬编码（90s/60s），对慢速模型不够
4. 流式场景只看总时长，不看空闲时间
```

## 修复方案

### 1. 游戏设置中添加超时配置

在 `游戏设置结构` 中添加：

```typescript
后台队列超时配置?: {
  变量生成超时秒: number;    // 默认 120
  地图更新超时秒: number;    // 默认 90
  文章优化超时秒: number;    // 默认 180
  动态世界超时秒: number;    // 默认 120
  规划分析超时秒: number;    // 默认 120
  流式空闲超时秒: number;    // 默认 60（流式场景无新数据到达的时长）
}
```

在 `默认游戏设置` 中提供合理的默认值。

### 2. 变量生成接口传递 signal

在 `variableCalibrationCoordinator.ts` 的 `执行变量校准并合并响应` 接口中添加 `signal?: AbortSignal` 参数。
如果传入了外部 signal，用 `AbortSignal.any([controller.signal, externalSignal])` 合并。

### 3. 后台队列使用用户配置的超时

在 `sendWorkflow.ts` 后台队列启动时：
```typescript
const 超时配置 = currentState.gameConfig.后台队列超时配置;
const 变量生成超时信号 = AbortSignal.timeout((超时配置?.变量生成超时秒 ?? 120) * 1000);
const 地图更新超时信号 = AbortSignal.timeout((超时配置?.地图更新超时秒 ?? 90) * 1000);
```

### 4. 流式空闲超时

在 `chatCompletionClient.ts` 的 SSE 解析中，检测"无新数据到达"时长：
```typescript
let lastChunkAt = Date.now();
const IDLE_TIMEOUT = 流式空闲超时秒 * 1000;
if (Date.now() - lastChunkAt > IDLE_TIMEOUT) {
    throw new Error('Stream idle timeout');
}
```

### 5. 手动触发不受限

手动触发（如 `解析地图` 按钮、`重新生成开局`）不走 `执行可重试独立阶段`，
而是直接调用底层函数，不传超时 signal。

## 默认超时值

| 阶段 | 默认值 | 理由 |
|------|--------|------|
| 变量生成 | 120s | JSON 解析+生成，中等耗时 |
| 地图更新 | 90s | 增量更新，较轻量 |
| 文章优化 | 180s | 正文润色，可能较长 |
| 动态世界 | 120s | COT 思考+16步输出 |
| 规划分析 | 120s | 切章判断，中等耗时 |
| 流式空闲 | 60s | 流式场景无新数据到达 |

## 验证

1. 默认值下所有阶段正常完成
2. 设置超时为 1 秒，确认超时后阶段被跳过，后续阶段继续
3. 手动触发单个步骤不受超时影响
4. 流式空闲超时：模拟 AI 暂停输出，确认超时触发
