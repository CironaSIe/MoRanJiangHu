## 四、关键风险与缓解

| 风险 | 级别 | 影响 | 缓解 |
|------|------|------|------|
| 记忆系统命令丢弃修复后调用方漏传参数 | 🔴 | 编译失败 | TypeScript 强制所有调用方更新（3个调用方：responseCommandProcessor.ts:1469、planningUpdateWorkflow.ts:254、runtimeVariableWorkflow.ts:325） |

| 数组合并策略改变导致存档体积增大 | 🟡 | 存档膨胀 | 设置数组最大长度限制，超出后丢弃最旧条目 |
| 深拷贝统一后 Date 类型处理 | 🟡 | 时间字段错误 | 提取共享 deepClone 工具，增加 Date 特殊处理 |

| GameSettings.tsx useEffect 覆盖本地编辑 | 🟡 | 用户编辑丢失 | 需确认是否有外部重置表单的合法场景 |

## 十二、叙事平静值 & 性别比例演变——创意工坊完善

### 背景

叙事平静值（Pressure 分支）和性别比例演变（Gender 分支）已合并到 Merged 分支，
游戏内设置（GameSettings.tsx）已有基础界面，但**创意工坊（CreativeWorkshopModal.tsx）
缺少这两个功能的配置入口**。

现有数据流：
- **性别比例演变**：`ModeRuntimeProfile.性别比例演变预设` → 游戏设置 `性别比例自动演变`
  → `worldEvolutionWorkflow` 读取（`modeRuntimeProfile` 优先，fallback 到 `gameConfig`）
  - 创意工坊已有 `npc.genderRatio`（男女比例），但**缺少** `性别比例演变预设` 开关
- **叙事平静值**：`游戏设置.叙事平静值配置` → `sendWorkflow` / `worldEvolutionWorkflow` 读取
  - 配置**不在** `ModeRuntimeProfile` 中，创意工坊完全无法控制
  - 游戏设置没有创意工坊锁定机制（不像性别比例演变有 `性别比例演变预设` 锁定）

### 修改目标

创意工坊能完整控制两个功能的全部参数，游戏设置作为全局默认值，创意工坊题材模式可以覆盖。

### 修改范围

#### 1. `models/system.ts` — ModeRuntimeProfile 新增字段

```typescript
// ModeRuntimeProfile 接口新增：
叙事平静值配置?: 叙事平静值配置结构;  // 题材模式可覆盖的叙事平静值配置
```

**副作用分析**：
- `ModeRuntimeProfile` 是可选字段扩展，不影响现有存档（`?.` 可选链）
- 需要同步更新 `规范化模式运行时配置()` 中的规范化逻辑

#### 2. `utils/modeRuntimeProfile.ts` — 默认值规范化

在 `规范化模式运行时配置()` 中新增 `叙事平静值配置` 的规范化：

```typescript
叙事平静值配置: 规范化叙事平静值配置(raw?.叙事平静值配置, official.叙事平静值配置)
```

**副作用分析**：
- 复用 `gameSettings.ts` 中已有的 `规范化叙事平静值配置()` 函数
