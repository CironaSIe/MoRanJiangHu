### 2.2 初始化流程

**`useGameState.ts:294-360`** — 启动时从 IndexedDB 加载设置：

```
组件挂载
  → 检查存档数量 (dbService.读取存档数量())
  → 迁移图片资源到独立存储 (仅非原生环境)
  → 预热图片资源缓存
  → 读取主题 → 读取 API 配置 → 读取提示词池
  → 读取节日配置 → 读取视觉设置 → 读取图片管理设置
  → 读取游戏设置 → 读取记忆设置
  → 标记主题持久化就绪 (防止 'day' 默认值覆盖用户设置)
```

### 2.3 关键设计特点

- **30+ 独立 useState**：无 useReducer 或 Context，所有状态平铺在单一 hook 中
- **惰性提示词加载**：`prompts加载PromiseRef` 确保只加载一次，配合 `requestIdleCallback` 延迟加载
- **主题保护机制**：`主题持久化就绪Ref` 防止首次挂载时内存默认值 `'day'` 覆盖 IndexedDB 中已保存的主题
- **Ref 持有 AbortController**：4 个 Ref 分别用于主请求、回忆、变量生成的中断控制

### 4.4 图片资源管理

**存储策略**: DataUrl → IndexedDB `image_assets` 存储 → 返回 `wuxia-asset://` 引用

```
保存图片资源(dataUrl)
  ├─ 检查签名缓存 (已存在 → 直接返回引用)
  ├─ 写入 image_assets 存储 {id, dataUrl, createdAt}
  ├─ 注册内存缓存
  ├─ 加入延迟上传队列 (返回主界面时批量上传图床)
  └─ 返回 wuxia-asset:// 引用

读取图片资源(refOrId)
  ├─ 解析引用 ID
  ├─ 从 image_assets 读取
  └─ 更新内存缓存 + 签名缓存
```

**缓存预热** (`dbService.ts:872-919`):
- 最多预热 **24 张** 图片 (`图片缓存预热最大条目数`)
- 总字符上限 **18MB** (`图片缓存预热最大字符数`)
- 倒序读取（最新的优先）

2. **枚举值**: 白名单校验，不匹配则用 fallback
3. **数值约束**: `规范化字数要求` → `Math.max(50, ...)`
4. **嵌套对象**: 递归规范化 (DeepSeek策略, GLM策略, 独立APIGPT模式)
5. **旧版兼容**: `主剧情消息模式` 中 `'Gemini默认'` / `'默认'` → `'Gemini模式'`
6. **酒馆预设**: 规范化列表 → 匹配当前选中预设 → 级联更新预设对象/角色ID/名称

**关键默认值** (`默认游戏设置`, `gameSettings.ts:209-265`):
- 字数要求: 1500
- 叙事人称: 第二人称
- 主剧情消息模式: Gemini模式
- 启用回合结束自动存档: true
- 启用女主剧情规划: true
- 剧情风格: 一般

**`导出ZIP存档文件()`** (`saveArchiveService.ts:267-318`):

```
导出ZIP存档文件(options?)
  ├─ 获取存档列表 (options.saves 或 dbService.导出存档数据())
  ├─ 遍历每个存档:
  │   ├─ 深拷贝存档

  │   │   ├─ 存入 ZIP: 图片/{saveKey}/{fileName}.{ext}
```

## 九、关键设计决策总结

| 设计点 | 决策 | 理由 |
|--------|------|------|
| **状态管理** | 30+ 独立 useState，无 Redux/useReducer | 简单直观，符合项目规模 |
| **存档写入** | 串行队列 + 签名去重 | 防止并发写入冲突和重复存档 |
| **图片存储** | DataUrl → IndexedDB → 引用 ID | 避免存档 JSON 过大，支持图床迁移 |
| **设置版本** | `version: 2` 字段 | 未来设置格式升级时可做迁移 |
| **存档谱系** | `存档系列ID` + `存档谱系版本` | 支持回档链追踪和云端同步去重 |
| **轻量视图** | `投影存档谱系轻量视图()` | 避免加载完整存档到内存，防止移动端 OOM |
| **主题保护** | `主题持久化就绪Ref` | 防止 React 初始默认值覆盖用户已保存值 |
| **自动存档上限** | 最多 5 个 | 平衡回档灵活性和存储占用 |
| **延迟图床上传** | 返回主界面后批量上传 | 不阻塞存档主流程 |
| **设置规范化** | 白名单 + fallback 链 | 保证旧版存档/设置在新版仍能正常工作 |
