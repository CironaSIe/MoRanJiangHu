# 战斗与地图系统机制分析

## 一、战斗系统（BattleModal）

### 1.1 战斗流程

战斗系统以 `BattleModal` 组件为核心，采用**信息展示型**设计——不执行战斗逻辑运算，而是将 AI 返回的战斗状态结构化呈现给玩家。

**入口条件：**
- 角色数据 (`character`) + 战斗状态 (`battle`) 由 AI 响应中的命令写入
- `battle.敌方` 数组描述敌方单位列表
- `battle.是否战斗中` 控制战斗/休战状态显示

**核心数据流：**
```
AI 响应 → responseCommandProcessor → 写入战斗状态 → BattleModal 展示
```

### 1.2 战斗指标体系

系统为每个战斗单位计算七项核心指标（`计算战斗指标` 函数）：

| 指标 | 计算公式 | 含义 |
|------|---------|------|
| **攻势** | `baseAttack + 力量×1.2 + 内力×0.08 + 境界×3 + 精力比×10 - 毒性×0.35` | 每回合伤害压力 |
| **近战伤害** | 等于攻势 | 贴身输出 |
| **远程物理伤害** | `攻势 × remoteFactor` (0.55~0.9) | 暗器/弓弩/飞剑 |
| **法术伤害** | `基础伤害 + 属性加成 + 内力×系数` | 法术/内功攻击 |
| **近战守势** | `防御 + 体质×0.7 + 根骨×0.5 + 装备物理×0.9 + 装备近战 + 气血比×8 - 毒性×0.25` | 近战减伤 |
| **远程物理守势** | `防御×0.85 + 体质×0.45 + 装备物理×0.75 + 掩体 + 气血比×6` | 远程减伤 |
| **法术守势** | `根骨×0.9 + 内力×0.08 + 装备法术 + 境界×2 + 气血比×5` | 法术减伤 |
| **身法** | `敏捷×1.2 + 精力比×16 - 毒性×0.45` | 回合移动预算 |
| **续航** | `气血比×45 + 精力比×45 + min(10, 内力/10) - 毒性×0.25` | 持续作战能力 |

### 1.3 伤害类型三分法

伤害被拆分为三个独立通道：
- **近战物理伤害**：贴身攻击，受近战守势抵抗
- **远程物理伤害**：弓弩/暗器/飞剑，受远程物理守势 + 掩体抵抗
- **法术伤害**：内功/法术/咒符，受法术守势抵抗

### 1.4 装备防护系统

`读取装备防护` 函数遍历单位已装备物品，根据耐久度比率折算防护值：
- **物理防御**：所有装备的 `物理防御 × 耐久比`
- **法术防御**：所有装备的 `内功防御 × 耐久比`
- **掩体加成**：名称含"盾/甲/铠/护"的装备额外 +6
- **近战格挡**：名称含"格挡/招架"的装备额外 +4

### 1.5 高度修正

战场地形高度影响伤害输出（`计算高度修正`）：
- **远程倍率**：`1 + 高度差×0.18`（高处打低处增益，反之减益），范围 0.65~1.9
- **近战倍率**：`1 + 高度差×0.06`，范围 0.78~1.35
- 高度由上下文文本推断（高处/坡/崖 → +1.2；水/泥/谷 → -0.8）

### 1.6 前后排机制

`自动划分前后排` 根据单位 X 坐标自动判定：
- 以双方最前端单位为基准线
- 距前线 ≤ 7 单位为"前排"，其余为"后排"
- 越位攻击需满足：身法差 ≥ 距离差，否则触发借机攻击

### 1.7 毒性影响

毒性是一个全维度减益：
- 来源：状态/异常状态/技能描述中的关键词（毒/中毒/淬毒/麻痹/致盲/腐蚀）
- 每层毒性降低：攻势×0.35、身法×0.45、近战守势×0.25
- 上限 30 层

## 二、地图六层树结构

### 2.1 层级定义

地图采用严格的六层树状结构（`地图层级类型`）：

```
寰宇 (深度0) → 大地点 (深度1) → 中地点 (深度2) → 小地点 (深度3) → 区地点 (深度4) → 子地点 (深度5)
```

| 层级 | 语义 | 默认网格尺寸 | 显示名 |
|------|------|------------|--------|
| 寰宇 | 诸天万界/位面 | 64×48 | 寰宇 |
| 大地点 | 国家/大陆/秘境 | 48×48 | 大地点 |
| 中地点 | 城市/宗门/坊市 | 36×36 | 中地点 |
| 小地点 | 街区/院落/村庄 | 28×28 | 小地点 |
| 区地点 | 建筑/地标/客栈 | 24×24 | 建筑 |
| 子地点 | 房间/室内空间 | 24×24 | 房间 |

### 2.2 DT-xxx 节点 ID

每个地图节点分配唯一 ID，格式为 `DT-三位序号`（如 `DT-001`、`DT-042`）：
- 序号全局递增，从已有最大序号接续
- 新节点通过 `nextId()` 分配，跳过已用 ID
- 重建时旧 ID 仅用于名称映射，不强制复用

### 2.3 节点数据结构

```typescript
interface 地图层级结构 {
    ID: string;              // DT-xxx
    名称: string;
    层级: 地图层级类型;
    描述: string;
    控制势力?: string;       // 势力归属
    势力影响?: string;
    势力标签?: string[];
    归属: { 大地点, 中地点, 小地点 };
    父级ID?: string;
    锚点坐标: { x, y };      // 在父层中的位置
    网格宽度, 网格高度;
    边界四角坐标: [4点];
    建筑物ID列表, 道路ID列表, 人物ID列表;
}
```

### 2.4 别名兼容

旧存档中的非标准层级名会被自动映射：
- "具体地点" → 区地点
- "室内"/"房间"/"建筑"/"房屋" → 子地点
- "地点"/"区域" → 区地点

## 三、地图空间系统（mapSpatial.ts）

### 3.1 核心入口

`补齐世界地图空间字段(world, options)` 是地图空间的顶层入口，执行以下流程：

```
规范化地图层级列表 → 规范化地图建筑列表 → 规范化地图道路列表 → 规范化地图人物列表
    → 从旧版字段派生地图空间 → 按层级补齐道路 → 去重地图人物落点 → 验证并修复地图数据
```

### 3.2 A* 寻路算法

`规划避让建筑正交路径` 实现了基于曼哈顿距离的 A* 算法：

1. **障碍建模**：将建筑四角坐标扩展 0.9 格后栅格化为障碍集合
2. **最近空闲单元格**：若起点/终点落在障碍内，使用 BFS 向外搜索最近的空闲格
3. **A* 搜索**：
   - 启发函数：曼哈顿距离 `|dx| + |dy|`
   - 移动方式：四方向正交（上下左右）
   - 代价：每步代价为 1
4. **Fallback**：若 A* 找不到路径，使用 L 形路径（先 X 后 Y 或先 Y 后 X）
5. **路径简化**：`简化正交路径` 移除共线中间点

### 3.3 BFS 最近空闲单元格

`nearestFreeCell` 函数使用广度优先搜索：
- 从目标点出发，四方向扩展
- 使用 `visited` 集合避免重复
- 返回第一个不在障碍集合中的格子

### 3.4 建筑布局优化

`避让层级建筑重叠` 确保建筑不重叠：
1. 按中心坐标排序（先 Y 后 X）
2. 对每个建筑与已放置建筑做 AABB 碰撞检测（含 0.45 间隙）
3. 碰撞时按网格排列重新定位（最多 200 次尝试）

### 3.5 道路生成策略

道路按层级类型采用不同策略：

| 层级类型 | 道路策略 |
|---------|---------|
| 聚落层级（城镇/村庄） | 横纵网格：2-4 条横向 + 2-4 条纵向道路 |
| 野外层级 | 单主路贯穿（保留"官道/山道"等主路，删除多余巷道） |
| 室内层级 | 无道路 |
| 其他层级 | 每个建筑生成接入路连接到最近道路 |

### 3.6 人物落点计算

`计算人物落点` 根据位置描述选择落点策略：
1. **近身位置**（身边/旁边/附近）：在参照点附近偏移
2. **建筑匹配**：位置文本命中建筑名称 → 建筑中心偏移
3. **野外位置**：使用正弦波模拟自然分布
4. **默认**：建筑入口附近的道路旁

`计算人物避让落点` 确保人物间距 ≥ 1.25，使用角度扩散算法（最多 16 次尝试）。

## 四、NPC 地图定位（mapNpcLocation.ts）

### 4.1 精确位置匹配算法

NPC 位置匹配分为两个层次：

**广义位置片段**（`提取NPC地图位置片段`）：
- 读取字段：`位置路径`、`当前位置`、`当前地点`、`所在地点`、`所在位置`、`具体地点`、`地点`、`位置`、`归属.具体地点`、`归属.小地点`、`归属.中地点`、`归属.大地点`

**精确位置片段**（`提取NPC精确地图位置片段`）：
- 排除粗粒度的 `归属.小地点/中地点/大地点`
- 仅保留具体位置字段

### 4.2 文本匹配算法

`地图文本相互命中` 使用归一化包含匹配：
1. 去除所有空白字符 + 转小写
2. 完全相等 → 命中
3. 任一字符串长度 ≥ 2 且互相包含 → 命中

### 4.3 节点匹配优先级

`选择NPC精确匹配地图节点` 的匹配顺序：
1. **精确相等**：节点名称归一化 == NPC 位置片段归一化
2. **模糊包含**：节点名称与 NPC 位置片段互相包含

`选择NPC匹配地图节点`（广义）的额外回退：
3. **当前叶子节点回退**：若无精确位置且 `是否在场 === true`，归入当前具体地点

### 4.4 视图判定

`NPC属于地图视图` 判断 NPC 是否应显示在当前地图视图中：
- 优先使用节点匹配
- 若无匹配，检查 NPC 位置是否命中视图路径名称
- 最后回退：NPC 在场 + 视图名称包含当前环境名称

## 五、地图渲染

### 5.1 RegionMap.tsx — 区域地图

`RegionMap` 是左侧主地图组件，根据层级类型使用不同的渲染模式：

| 层级 | 渲染模式 | 视觉特征 |
|------|---------|---------|
| 寰宇 | 星空图 | 深蓝背景 + 200 颗星 + 椭圆轨道 + 星球标记 |
| 大地点 | 大陆板块 | 力导向多边形 + 碰撞避免 + 色板区分 |
| 中地点 | 驿道路网 | 最小生成树(MST)连接城寨 + 双线古驿道 |
| 小地点 | 网格街区 | 横纵道路 + 彩色建筑方块 + 空位预留 |
| 区地点/子地点 | 房间卡片 | 矩形房间块 + NPC 标记 + 描述文字 |

**交互功能：**
- 滚轮缩放（1x ~ 8x）
- 拖拽平移（PointerEvent + setPointerCapture）
- 点击节点导航到子层级
- 定位按钮：缩放到当前节点 4x

**布局算法：**
- 寰宇/大地点：力导向布局 + 碰撞检测
- 中地点：网格布局 + MST 道路
- 小地点：固定网格 + 名称哈希打乱分配
- 房间：居中卡片式布局

### 5.2 GridMapScene.tsx — 网格地图场景

`GridMapScene` 是右侧详细信息面板中的 SVG 地图渲染器：

**渲染层次：**
1. 背景（绿地/山体/水体）
2. 网格线（每 4 格加粗）
3. 等高线（正弦波模拟地形）
4. 道路线（双线：实线 + 虚线）
5. 建筑面（四角多边形 + 名称标签）
6. 人物点（圆形标记 + 姓名标签）

**人物标签布局：**
- 使用 8 个候选偏移方向
- 评分函数考虑：重叠惩罚 + 距其他点距离 + 边界惩罚
- 主角和重要单位优先布局
- 标签不可见时隐藏（评分 ≥ 1000）

**防幽灵 NPC 过滤：**
- 只显示主角 + 社交面板中已存在的 NPC
- 通过姓名/ID 双重匹配验证

### 5.3 LocationBrowser.tsx — 地点浏览器

`LocationBrowser` 是地图面板的容器组件，提供三栏布局：

**左侧：区域地图**
- 面包屑导航（可点击跳转）
- 返回上一层按钮
- 回忆解析按钮（触发 memory_regenerate）

**右侧：三栏固定面板**
1. **地点索引**：递归树形组件 `LocationTreeItem`
   - 自动展开当前路径
   - 叶子节点显示在场 NPC（彩色标签）
   - 支持展开/收起（最多 30 层深度）
2. **区域介绍**：选中节点的详细信息
   - 控制势力/势力影响/势力标签
   - "前往此地"/"查看详情"快捷命令按钮
3. **回忆解析日志**：流式输出 AI 重建地图的过程

## 六、地图更新队列

### 6.1 两种更新模式

系统支持两种地图更新模式（`地图更新模式`）：

| 模式 | 触发方式 | 用途 |
|------|---------|------|
| `auto_incremental` | 正文后自动触发 | 增量更新：只新增本回合确认的地点 |
| `memory_regenerate` | 手动"回忆解析" | 全量重建：从回忆库提取地点重建整棵树 |

### 6.2 增量更新流程（auto_incremental）

```
正文响应完成 → 地图更新队列阶段 → 调用 AI → 解析命令 → 应用 push/set/delete
```

**AI 提示词规则：**
1. 只在本回合正文确认新地点时才输出 push 命令
2. 同名地点不重复 push
3. 父级 ID 优先填写已有 DT-xxx ID，否则填写名称
4. 无新增时输出"无"
5. 强制写入势力信息（控制势力/影响/标签）

**命令格式：**
```
push 世界.地图层级 = {"名称":"...","层级":"区地点","父级ID":"DT-004","描述":"..."}
```

**解析器（`解析地图自动更新命令`）：**
- 提取 `<命令>` 块内容
- 逐行解析 push/set/delete 动作
- 过滤重复名称
- 规范化层级类型
- Fallback：若命令解析为空，尝试解析为 JSON 地点树（全量替换）

### 6.3 回忆库重建流程（memory_regenerate）

**触发路径：**
```
用户点击"回忆解析" → App.tsx handleRegenerateMapFromMemory → 生成地图更新(memory_regenerate)
    → AI 返回 JSON 地点树 → 构建地图层级替换结果 → 清空旧地图 → 写入新地图 → 自动保存
```

**回忆库线索构建（`构建回忆库地图线索`）：**
- 取最近 120 条回忆档案（概括 600 字 + 原文 1800 字）
- 长期记忆（每条 900 字）
- 中期记忆（最近 80 条，每条 700 字）
- 短期记忆（最近 80 条，每条 500 字）
- 即时记忆（最近 40 条，每条 900 字）

**AI 输出格式：**
```json
{"地点树":[{"名称":"...","层级":"...","父级ID":"父级名称或ID","描述":"...","控制势力":"..."}]}
```

**重建规则：**
- 根节点必须是 `层级:"寰宇" 名称:"诸天万界"`
- 旧地图在写入前被完全清除
- 不复制旧层级数据
- 不生成坐标/道路/建筑等旧字段
- 生成新的 DT-xxx ID 序列

### 6.4 API 配置

- `memory_regenerate` 使用地图生成接口配置（`获取地图生成接口配置`）
- `auto_incremental` 使用自动更新接口配置（`获取地图自动更新接口配置`）
- 独立 API 未配置时回退到主 story API
- 温度：增量 0.35（保守），重建 0.7（创造）

## 七、地图布局优化器（mapLayoutOptimizer.ts）

### 7.1 设计思路

布局优化器参考真实城镇设计，生成"卷轴节点图"风格的布局：
- 当前区域居中
- 关键建筑环绕中心
- 路线从中心放射到各建筑

### 7.2 触发条件

```typescript
if (!是否聚落层级(层级) || 建筑列表.length < 4) {
    return { 建筑: 建筑列表, 道路: [] };  // 不优化
}
```

只对聚落层级（名称含"镇/城/坊/市/街/巷/村/庄/院/宅/府/馆/楼/铺"）且 ≥ 4 个建筑生效。

### 7.3 卷轴环形建筑布局

`生成卷轴环形建筑布局`：
1. 计算中心点 `(width×0.5, height×0.52)`
2. 椭圆半径：`radiusX = min(width×0.34, max(width×0.24, 8))`
3. 从 -120° 开始等角度分布建筑
4. 建筑尺寸由名称哈希决定（宽度 3.1~4.8，高度 2.5~3.8）

### 7.4 卷轴连线路径

`生成卷轴连线路径`：
- 为中心到每个建筑生成放射状连线
- 最多 6 条路线（`slice(0, 6)`）
- 路线名称格式：`卷轴路线01`、`卷轴路线02`...
- 入口点计算：建筑朝向中心的一边中点

### 7.5 聚落层级判定

`是否聚落层级` 使用关键词匹配：
- 名称/描述含：镇、城、坊、市、街、巷、村、庄、院、宅、府、馆、楼、铺、坊市、客栈
- 排除室内层级和野外层级
- `小地点` 层级无条件视为聚落

## 九、关键设计决策总结

1. **地图与渲染分离**：数据层（六层树 + 空间字段）与渲染层（SVG 组件）完全解耦，同一份数据支持多种可视化方式。

2. **AI 驱动更新**：地图节点完全由 AI 生成，系统只负责解析、验证、补全空间字段，不硬编码地点逻辑。

3. **防御性空间计算**：mapSpatial.ts 包含大量兜底逻辑（建筑重叠避让、路径穿建筑、NPC 幽灵过滤），确保 AI 生成的不完美数据仍可有效渲染。

4. **双模式更新**：增量更新避免过度重建，回忆库重建解决旧存档迁移，两者互补。

5. **NPC 位置驱动显示**：地图节点不存储"在场人物"，NPC 显示完全由社交档案中的位置字段驱动，保证数据一致性。
