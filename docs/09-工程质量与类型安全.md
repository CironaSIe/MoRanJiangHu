# 工程质量与类型安全审计

## 一、严重问题（Critical）

### 1.1 动态代码执行 — `new Function()` 注入风险

**文件**: `services/ai/storyResponseParser.ts:1452`

```typescript
const result = new Function(`return (${trimmed})`)() as number;
```

虽然用正则 `/^[0-9+\-*/ .()]+$/` 做了白名单校验，但 `new Function()` 本质上是 `eval` 的变体。如果正则逻辑被修改或绕过（Unicode 混淆、正则引擎差异），将导致任意代码执行。

**建议**: 使用纯算术表达式解析器或手写递归下降解析器替代。

---

### 1.2 `dangerouslySetInnerHTML` — XSS 风险

**文件**: `components/features/Chat/MessageRenderers.tsx:1243`

```tsx
dangerouslySetInnerHTML={{ __html: htmlContent }}
```

虽然清洗链路存在（DOMPurify + 沙箱 iframe），但 DOMPurify 调用不够显式。AI 生成的 HTML 内容可能包含精心构造的绕过载荷。

---

### 1.3 提示词注入（Prompt Injection）防护缺失

**文件**: `hooks/useGame/sendWorkflow.ts:1259-1260`

```typescript
role: 'user',
content: sendInput,  // 玩家原始输入直接进入 AI prompt
```

玩家输入被直接作为 `user` 角色消息发送给 AI，没有过滤、转义或隔离机制。恶意玩家可以注入指令覆盖系统 prompt、伪造命令标签等。

---

### 2.4 TSC 类型检查错误（部分）

### 2.6 竞态条件

多个异步操作（故事生成、变量更新、地图更新、规划分析）并发执行：
- 用户快速连续发送消息时，后一次请求可能先于前一次完成
- `AbortController` 可取消请求，但已写入的状态不会自动回滚
- `Promise.all` 中某个失败不会回滚其他已完成的副作用

## 三、中优先级问题（Medium）

### 3.1 Token 溢出防护不足

只限制输出 token（最大 8192），没有限制输入上下文 token。长对话场景下（100+ 回合），上下文窗口必然溢出。

### 3.2 IndexedDB 无版本迁移

`onupgradeneeded` 只处理首次创建，没有版本迁移逻辑。

### 3.3 大列表无虚拟滚动

聊天记录、物品列表、NPC 列表全部普通 `.map()` 渲染。

### 3.4 云同步无冲突解决

last-write-wins 策略，多设备同时编辑可能丢失数据。

### 3.5 图片加载策略不完整

大部分 `<img>` 缺少 `loading="lazy"` 和 `onError` 回调。

---

4. 云同步冲突解决 — OT 或 CRDT
