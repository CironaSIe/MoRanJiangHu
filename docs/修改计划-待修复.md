## 第一优先级：真正的 Bug（必须修复）

### 🔴 B7: 后台队列阶段无超时保护
**详见**: `docs/修改计划-后台队列超时机制.md`

**问题**: `执行变量生成并合并响应` 和 `执行地图更新` 接口不接收 `signal` 参数，无法被取消或超时。AI 端点挂起时整个后台队列卡死。

**影响**: 玩家每回合发送消息后，如果变量生成 API 无响应，UI 永远显示"后台处理中"，后续阶段（动态世界、规划分析、地图更新）永远不执行。

**修复**: 详见 `docs/修改计划-后台队列超时机制.md`，需要：
1. 接口添加 `signal` 参数
2. 后台队列创建带超时的 signal
3. 超时后自动 skip 阶段

## 第二优先级：性能瓶颈（应当修复）

### 🟡 P5: 自动存档阻塞主线程
**位置**: `hooks/useGame/saveCoordinator.ts:604-720`（`创建存档数据`）

**问题**: 每次自动存档执行 ~20+ 次深拷贝（角色 50k 节点、社交 70k 节点），同步阻塞主线程 200ms-2s。

**实际影响**:
- 中型存档（50 回合/30 NPC）：卡顿 200-500ms
- 大型存档（200+ 回合/100+ NPC）：卡顿 500ms-2s
- 玩家可感知周期性卡顿

**根因**: `构建自动存档签名` 和 `创建存档数据` 各自独立规范化环境和记忆系统，重复执行。

**修复方案**:
1. 消除重复规范化（签名构建阶段的结果复用）
2. 用 `structuredClone` 替代手写 `deepClone`（原生更快）
3. 增量签名：只比较最后一条消息 digest，不全量规范化记忆系统

## 第三优先级：低风险问题（可后续处理）

### D6: 切换 baseMode 静默重置整个 profile
**位置**: `components/features/Workshop/CreativeWorkshopModal.tsx:926-933`

**实际影响**:
- **触发面窄**：只有贡献者（极少数高级玩家）操作
- **低频**：baseMode 是初始选择，正常流程不反复切换
- **丢失范围大**：整个 modeRuntimeProfile 被替换（13 个分区全部丢失）
- **无恢复机制**：没有确认、没有撤销

**修复方案**: 切换前加 `window.confirm` 确认对话框即可。成本低，足够应对。

## 确认为无需修复

### D7: 创意工坊 number 字段无 min/max 约束
**分析结果**: 消费方 `规范化叙事平静值配置` 有完善的 fallback（负数/0/非有限值 → 默认值）。唯一能穿透的是极大正整数，但只影响叙事平静值触发时机，不会崩溃。**低风险，无需修复。**

### D8: 发布模块无 number 字段校验
**分析结果**: 同 D7，消费方正则化会修正非法值。**低风险，无需修复。**

### P3: 大列表无虚拟化
**分析结果**: ChatList 已有 10 回合窗口化（`renderCount = 10`），全量渲染需用户主动点击"展开"。**已有轻量优化，无需立即修复。**

### P4: 40+ useState 导致全量重渲染
**分析结果**: React 18 自动批处理已合并同事件循环内的 setState。随功能增长会恶化，但属于架构级重构，**不适合在当前阶段做。**
