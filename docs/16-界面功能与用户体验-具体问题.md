# 界面功能与用户体验——具体问题

## 1. ApiSettings.tsx — API 设置

### 问题 1.1：删除配置无二次确认 🔴
**文件**: `components/features/Settings/ApiSettings.tsx:341-355`
```typescript
const handleDeleteActive = () => {
    if (!activeConfig) return;
    setForm((prev) => {
        const nextConfigs = prev.configs.filter((cfg) => cfg.id !== activeConfig.id);
        ...
    });
    setMessage('配置已删除。');
};
```
删除包含 apiKey/baseUrl 的配置瞬间完成且不可恢复。与 SaveLoadModal 删除存档的多步确认形成鲜明对比。

### 问题 1.2：测试连接期间可双击并发
**文件**: `components/features/Settings/ApiSettings.tsx:378-386`
`testingConnection` 在函数体内部才设为 true（异步状态更新），快速双击时防重入失效。

### 问题 1.3：message 状态多操作共用
**文件**: `components/features/Settings/ApiSettings.tsx:185`
所有操作共用一个 `message` 字符串。`handleTestConnection` 的 `finally` 中 `setMessage('')` 会清空之前所有有意义的消息。

### 问题 1.4：自定义 Token 输入无防抖
**文件**: `components/features/Settings/ApiSettings.tsx:611-633`
每次按键触发 `updateActiveConfig` → `setForm` → 整个组件重渲染。

### 问题 1.5：切换配置时不重置已选模型
**文件**: `components/features/Settings/ApiSettings.tsx:518-522`
切换配置时清空 `mainModelOptions`，但 `主剧情解析模型` 已从 `activeConfig.model` 读取新值。模型下拉为空但实际已有值。

## 2. MapModelSettings.tsx — 地图设置

### 问题 2.1：保存按钮在页面底部无提示
**文件**: `components/features/Settings/MapModelSettings.tsx:349-353`
与 ApiSettings 顶部 sticky 保存栏不同，MapModelSettings 保存按钮在页面最底部。用户修改了上半部分设置后需滚动到底底才能保存。

### 问题 2.2：回忆解析进行中无视觉遮罩
**文件**: `components/features/Settings/MapModelSettings.tsx:125-143`
回忆解析会删除当前全部地图内容并重建（破坏性操作），但执行期间页面其他部分仍可交互。

### 问题 2.3：fetchModelsFromCurrentConfig 不兼容 MiMo 鉴权
**文件**: `components/features/Settings/MapModelSettings.tsx:76-81`
```typescript
const res = await fetch(url, { headers: { Authorization: `Bearer ${resolvedApiKey}` } });
```
ApiSettings 对 MiMo 使用 `api-key` 请求头，但 MapModelSettings 统一使用 `Authorization: Bearer`。MiMo token plan 用户无法获取模型列表。

## 3. NewGameWizard.tsx — 新局向导

### 问题 3.1：大量使用 alert()
**文件**: `components/features/NewGameWizard.tsx:384-386, 391-394, 1536-1537, 1559-1560, 1577-1578`
```typescript
alert(`当前难度"${当前难度设定.label}"的出身重 roll 次数已用完`);
alert(`当前属性总点数超过 ${worldConfig.difficulty.toUpperCase()} 难度上限，请先回收 ${Math.abs(remainingPoints)} 点。`);
```
项目已有 `requestConfirm` 自定义确认弹窗，但仍大量使用原生 `alert()`。阻塞 JS 执行线程，样式与古风主题不匹配。

### 问题 3.2：属性点校验失败无视觉引导
**文件**: `components/features/NewGameWizard.tsx:1548-1552`
校验失败时只弹出 alert，无高亮属性面板或红色提示边框。

### 问题 3.3：进度条不准确
**文件**: `components/features/NewGameWizard.tsx:818`
```typescript
const stepProgress = ((step + 1) / STEPS.length) * 100;
```
基于固定步骤数计算，但某些步骤根据条件跳过或显示不同内容。

## 4. SaveLoadModal.tsx — 存读档

### 问题 4.1：导入文件后清空 input 可能导致 change 事件异常
**文件**: `components/features/SaveLoadModal.tsx:694`
```typescript
e.currentTarget.value = '';  // ← 在 await requestConfirm 之前清空
if (!ok) { setTransferMessage('已取消导入。'); return; }
```
用户取消导入后再次选择同一文件，某些浏览器不触发 change 事件。

### 问题 4.2：大量使用 alert() 双重反馈
**文件**: `components/features/SaveLoadModal.tsx:327, 345, 352, 378, 388, 409, 481, 501, 539, 575, 611, 635, 734`
项目已有 `setTransferMessage` 在 Modal 内显示反馈，但所有错误处理仍额外弹出 `alert()`。用户需关闭弹窗才能看到 Modal 内反馈。

### 问题 4.3：存档保护开启时删除按钮仍可见
**文件**: `components/features/SaveLoadModal.tsx:415-419`
删除按钮只是样式变灰 + disabled，但仍占据布局空间。误触后弹出 alert 打断操作流。

### 问题 4.4：hydrateSummary 依赖 transferMessage 导致意外重触发
**文件**: `components/features/SaveLoadModal.tsx:150`
`transferMessage` 变化触发整个 hydrate effect 重新运行，hydrate 过程被意外中断或延迟。

## 5. LandingPage.tsx — 首页

### 问题 5.1："本地游玩"按钮无子步骤指示
**文件**: `components/features/LandingPage.tsx:813-816`
主 CTA 按钮点击后弹出二级选择（新游戏/加载存档），但按钮无视觉暗示表明还有子选项。

### 问题 5.2：在线人数图表 localStorage 写入失败静默忽略
**文件**: `components/features/LandingPage.tsx:132-151, 668`
localStorage 写入失败时静默忽略，无用户提示。隐私模式或存储满时每次刷新都看到空白图表。

### 问题 5.3：全屏按钮无状态指示
**文件**: `components/features/LandingPage.tsx:745-753`
按钮文本始终显示"全屏"，无论当前是否已进入全屏模式。无 active 状态样式变化。

## 6. TopBar.tsx — 顶部栏

### 问题 6.1：hover 展开面板经过间隙时关闭
**文件**: `components/features/TopBar.tsx:620-627`
`TopItem` 的 `onMouseLeave` 将 `expandedType` 设为 null。`DetailCard` 和 `TopItem` 之间有 DOM 间隙（`mt-2`），鼠标快速移动时间隙触发面板关闭。

### 问题 6.2：移动端详情面板可能被视口裁剪
**文件**: `components/features/TopBar.tsx:603-612`
```typescript
className="left-full top-0 ml-1 mt-0"
panelClassName="w-[min(62vw,300px)] max-w-[300px]"
```
320px 宽度手机上，面板宽度 198px + 左侧按钮区域可能超出视口。无视口边界检测。

### 问题 6.3：历程天数无错误边界
**文件**: `components/features/TopBar.tsx:333-335`
```typescript
const derivedDayCount = useMemo(() => {
    return 计算游戏历程天数(parsedTime, parseCanonicalGameTime(游戏初始时间));
}, [parsedTime, 游戏初始时间]);
```
时间解析失败时返回负数或 0，顶部栏显示"第 0 天"或"第 -1 天"。

### 问题 6.4：移动端时间卡片信息拥挤
**文件**: `components/features/TopBar.tsx:559`
移动端把日期、时钟、历程天数挤在一行，窄屏上溢出或被截断。
