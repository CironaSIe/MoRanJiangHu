# 创意工坊运行时配置字段完整参考

> 本文档面向 AI 代理和开发者，描述创意工坊（CreativeWorkshopModal）中所有可配置字段的完整定义、默认值、行为逻辑和设计约束。

---

## 一、架构概述

### 数据流

```
创意工坊编辑（CreativeWorkshopModal）
  → ModeRuntimeProfile（内存对象）
    → 发布为创意工坊模块（JSON 存储）
      → 玩家选择题材模式
        → sessionLifecycleWorkflow：新建游戏时写入 gameConfig（存入存档）
          → 运行时消费（sendWorkflow / worldEvolution / systemPromptBuilder 等）
```

### 关键字段路径说明

- **嵌套路径**：如 `['npc', 'genderRatio']` 对应 `ModeRuntimeProfile.npc.genderRatio`
- **顶层字段**：如 `['性别比例演变预设']` 对应 `ModeRuntimeProfile.性别比例演变预设`
- **路径读写**：`读取运行时路径值(profile, path)` 用 `reduce` + `current?.[key]` 逐层读取；`写入运行时路径值` 深拷贝后逐层创建对象并赋值

### 字段类型与渲染

| 类型 | UI 组件 | 输入行为 | 存储格式 |
|------|---------|---------|---------|
| `text` | `<input>` | 单行文本 | `string` |
| `textarea` | `<textarea>` | 多行文本 | `string` |
| `list` | `<textarea>` | 以顿号/逗号/换行分割 | `string[]` |
| `record` | `<textarea>` | 每行 `键=值` | `Record<string, string>` |
| `bool` | `<checkbox>` | 开关 | `boolean` |
| `boolGroup` | 多个 `<checkbox>` | 一组开关 | `Record<string, boolean>` |
| `number` | `<input>` | 数值，非法值回退为 0 | `number` |
| `baseMode` | `<select>` | 题材模式选择，**切换会重置整个配置** | 题材模式类型 |
| `currencyMode` | `<select>` | 7 选项下拉 | `string` |
| `timeFormatMode` | `<select>` | 6 选项下拉 | `string` |
| `realmConfig` | `<textarea>` | JSON 编辑器 | `{ levelNames: string[]; parseRules: Array<{pattern, level}> }` |
| `currencySystemEditor` | `<CurrencySystemEditor>` | 可视化货币编辑器 | `CurrencySystem` |
| `currencySystemJson` | `<textarea>` | 原始 JSON 编辑 | `CurrencySystem` |

## 二、分区详述

### 1. 题材身份（identity）

> 定义题材模式的基础身份标识。**切换 `baseMode` 会重置整个配置**。

| 字段 | 路径 | 类型 | 默认值 | 说明 |
|------|------|------|--------|------|
| 模式 ID | `identity.modeId` | text | 题材 value（如 `wuxia`） | 唯一标识，自动生成，不可手动编辑 |
| 显示名 | `identity.displayName` | text | 题材 label（如 `武侠`） | 游戏内显示名称 |
| 继承官方基础模式 | `identity.baseMode` | baseMode | `wuxia` | **危险字段**：切换会触发 `创建默认模式元数据草稿()` 重置所有字段 |
| 现代题材 | `identity.isModern` | bool | 题材组决定 | urban_xianxia/modern/apocalypse/infinite 组为 `true` |
| 启用修炼 | `identity.usesCultivation` | bool | 题材组决定 | xianxia/urban_xianxia 组为 `true` |
| 末日题材 | `identity.isApocalypse` | bool | `group === 'apocalypse'` | 影响生存资源默认值 |
| 生存模式 | `identity.isSurvival` | bool | `isApocalypse \|\| isInfinite` | 影响资源计数器默认开关 |
| 同人/IP 模式 | `identity.isFandomIp` | bool | `false` | 同人模式启用原著事件链优先 |

**设计注意**：
- `baseMode` 变更时，`CreativeWorkshopModal` 会调用 `创建默认模式元数据草稿(nextMode)` 重置整个 `modeRuntimeProfile`，用户未保存的改动会丢失
- `isModern`、`usesCultivation`、`isApocalypse`、`isSurvival` 由 `baseMode` 自动推导，不应独立于 `baseMode` 修改

### 2. 经济系统（economy）

> 定义货币体系、市场口径和物品经济约束。

| 字段 | 路径 | 类型 | 默认值 | 说明 |
|------|------|------|--------|------|
| 货币系统模式 | `economy.__currencySystemMode` | 三按钮组 | — | **不持久化**，仅控制 UI 显示模式 |
| 货币显示 | `economy.currencyDisplayMode` | currencyMode | 题材预设 | 7 选项：wuxia/xianxia/fantasy/urban/modern/apocalypse/infinite |
| 上层货币名称 | `economy.currencyTiers.upperName` | text | 自动计算 | 如 `金`/`灵石`/`信用点` |
| 中层货币名称 | `economy.currencyTiers.middleName` | text | 自动计算 | 如 `银`/`灵晶`/`角` |
| 底层货币名称 | `economy.currencyTiers.lowerName` | text | 自动计算 | 如 `铜`/`灵屑`/`分` |
| 上转中汇率 | `economy.currencyTiers.upperToMiddleRate` | text | 自动计算 | 正整数 |
| 中转底汇率 | `economy.currencyTiers.middleToLowerRate` | text | 自动计算 | 正整数 |
| 可视化货币编辑器 | `economy.currencySystem` | currencySystemEditor | — | 新版动态货币系统，优先级高于旧版 |
| 高级 currencySystem JSON | `economy.currencySystem` | currencySystemJson | — | 直接编辑 JSON |
| 题材货币说明 | `economy.primaryCurrency` | textarea | 题材预设 | 约束 AI 货币口径 |
| 底层记账单位 | `economy.accountingUnit` | text | 自动计算 | 如 `铜钱`/`灵点` |
| 旧兼容换算说明 | `economy.exchangeRules` | textarea | 题材预设 | 旧版货币换算规则 |
| 市场名称 | `economy.marketName` | text | 题材预设 | 拍卖行名称 |
| 市场动词 | `economy.marketVerb` | text | 题材预设 | 市场行为描述 |
| 允许物品类型 | `economy.allowedItemTypes` | list | 题材生成 | 允许的物品类型 |
| 禁用关键词 | `economy.bannedKeywords` | list | 题材生成 | 经济系统禁词 |

**三种货币模式**：
1. **新版动态货币**（推荐）：使用 `currencySystem` 对象，支持单一/多层/自定义单位
2. **旧版三层货币**：使用 `currencyTiers`（upper/middle/lower + 汇率）
3. **高级 JSON**：直接编辑 `currencySystem` JSON

**优先级**：`currencySystem` 存在时优先使用新版，否则 fallback 到 `currencyTiers`。

### 10. 世界叙事（顶层字段）

> 控制世界演变的叙事机制。**这些字段不在任何子对象内，是 ModeRuntimeProfile 的顶层字段。**

| 字段 | 路径 | 类型 | 默认值 | 说明 |
|------|------|------|--------|------|
| 性别比例自动演变 | `性别比例演变预设` | bool | `false` | 是否允许世界演变自动调整性别比例 |
| NPC 男女比例 | `npc.genderRatio` | text | 题材生成 | （同 NPC 系统，此处重复显示） |
| 启用叙事平静值 | `叙事平静值配置.启用` | bool | `false` | 总开关，关闭时不注入任何推进文本 |
| 无标签增量 | `叙事平静值配置.无标签增量` | number | `2` | 无标签回合的平静值增量 |
| 延续增量 | `叙事平静值配置.延续增量` | number | `1` | 延续标签回合的平静值增量 |
| 上限 | `叙事平静值配置.上限` | number | `32` | 平静值上限，超过后强制触发新事件 |
| 最低触发阈值 | `叙事平静值配置.最低触发阈值` | number | `12` | 低于此阈值不注入推进文本 |
| 阈值文本 | `叙事平静值配置.阈值文本` | list | 6 段文本 | 从最低阈值到上限等分的推进文本 |

**叙事平静值机制说明**：

```
AI 输出 <情节事件> 标签：
  - 介入/结束/退出 → 归零
  - 延续 → +1
  - 无标签 → +2

当 平静计数 >= 最低触发阈值(12)：
  按 (上限-下限)/段数 等分，注入对应段落的推进文本

当 平静计数 >= 计算远处联动阈值()：
  在 world演变中优先推进远处事件（主角参与度 <= 0.2）
```

**阈值文本默认值（6 段）**：
1. `一切如常。`
2. `一切如常，有些细节可以留意。（1 个方向）`
3. `一切如常，远处似乎有些变化。（2 个方向）`
4. `太长时间没有变化了，或许该有些新的事。（3 个方向）`
5. `风平浪静得太久了，可以有些新的动向了。（4 个方向）`
6. `一切如常，只是似乎有什么事要发生了。`

**性别比例演变机制说明**：

```
覆盖优先级：创意工坊预设 > 游戏设置 > 题材默认(false)

生效链路：
  worldEvolutionWorkflow 读取 genderEvolutionEnabled
    → true: 注入 Step8.5（世界级性别比例审计）+ Step8.6（地点级性别比例审计）
    → false: 跳过性别比例更新

Step8.5 触发条件（全部满足）：
  模式A（一次性极大事件）：
    1. 正文已坐实改写世界机制的事件
    2. 因果链完整
    3. 是已发生事实
  模式B（渐进式变化）：
    1. 正文已坐实新法术/力量体系
    2. 该体系改写世界性别机制
    3. 世界机制改变已启动

绝对不触发（硬约束）：
  - 个体角色性转
  - 大规模集体性转（机制未变）
  - 模糊氛围描写
  - 暗示性表述
```

## 三、默认值来源

所有默认值按以下优先级确定：

```
创意工坊编辑时的初始值：
  1. 构建官方模式运行时配置(baseMode) — 基于选定题材的完整默认配置
  2. 规范化模式运行时配置(raw, baseMode) — 将用户编辑的值合并到默认值上

新建游戏时写入 gameConfig 的值：
  sessionLifecycleWorkflow.ts 中：
    预设键值 = {}
    如果 modeRuntimeProfile.性别比例演变预设 是 boolean → 写入 gameConfig.性别比例自动演变
    如果 modeRuntimeProfile.叙事平静值配置 是 object → 写入 gameConfig.叙事平静值配置
    规范化游戏设置({ ...prev, ...预设键值 }) — 规范化后写入
```

## 四、典型误区和建议行为

### 误区 1：切换 baseMode 会保留已编辑字段

**错误做法**：先编辑了经济系统的货币名称，然后切换 `baseMode`，期望保留已编辑的字段。

**实际行为**：切换 `baseMode` 会调用 `创建默认模式元数据草稿(nextMode)` 重置整个 `modeRuntimeProfile`，所有未保存的改动丢失。

**建议**：切换 `baseMode` 前提示用户确认；或在 UI 上明确标注"切换题材将重置所有配置"。

### 误区 2：叙事平静值阈值文本可以任意数量

**错误做法**：只设置 2-3 段阈值文本，但设置上限为 32、最低阈值为 12。

**实际行为**：分段公式为 `段宽 = (上限 - 下限) / 文本列表.length`。如果只有 2 段文本但范围是 20，每段覆盖 10 个计数单位，粒度太粗。

**建议**：阈值文本数量应与 `(上限 - 最低触发阈值)` 成比例。推荐每 3-5 个计数单位一段文本。默认 6 段覆盖 20 个单位（每段约 3.3）是合理的。

### 误区 3：性别比例演变预设设为 false 时仍可被游戏设置覆盖

**错误做法**：在创意工坊中将 `性别比例演变预设` 设为 `false`，期望游戏内用户可以通过游戏设置开启。

**实际行为**：`worldEvolutionWorkflow` 使用 `??` 运算符：`modeRuntimeProfile.性别比例演变预设 ?? gameConfig.性别比例自动演变 ?? false`。当预设为 `false` 时，`??` 不会穿透（只对 `null/undefined` 生效），所以游戏设置的值**不会**生效。同时 GameSettings.tsx 中 `性别比例演变预设 != null` 时开关被禁用。

**建议**：这是预期行为。如果希望用户可以在游戏中调整，创意工坊不应设置此字段（留 `undefined`），让游戏设置的 `性别比例自动演变` 生效。

### 误区 4：经济系统分区中 currencySystem 和 currencyTiers 同时设置

**错误做法**：同时填写了新版货币编辑器（currencySystem）和旧版三层货币名称（currencyTiers）。

**实际行为**：游戏优先使用 `currencySystem`，`currencyTiers` 仅在 `currencySystem` 不存在时作为 fallback。两者同时设置时，旧版配置被忽略。

**建议**：通过 UI 的 `currencySystemEditMode` 状态控制显示，避免用户同时看到两套配置。

### 误区 5：叙事平静值配置中的阈值文本为空数组

**错误做法**：将 `阈值文本` 清空（设为空数组 `[]`）。

**实际行为**：`systemPromptBuilder` 中处理了空数组情况——取最后一条文本或 fallback 到默认文本。`计算远处联动阈值` 中 `Math.max(1, 段数)` 保底为 1，不会除零。但 UI 中的"当前阈值文本"显示为"暂无"。

**建议**：不应清空阈值文本。如果不想注入推进文本，应将 `启用` 设为 `false`。

### 误区 6：在创意工坊中编辑嵌套对象的顶层字段

**错误做法**：直接修改 `ModeRuntimeProfile` 的顶层字段（如 `性别比例演变预设`），而不是通过 `modeRuntimeProfile.npc.xxx` 子对象。

**实际行为**：`读取运行时路径值` 和 `写入运行时路径值` 都支持任意深度的路径数组。顶层字段的路径就是 `['性别比例演变预设']`，子对象字段的路径是 `['npc', 'genderRatio']`。

**建议**：确保路径数组正确。顶层字段路径长度为 1，子对象字段路径长度 >= 2。

### 误区 7：number 类型的字段传入非数值

**错误做法**：在创意工坊中将 `无标签增量` 填入 `"abc"` 或留空。

**实际行为**：`更新运行时配置字段` 中 `number` 类型的处理是 `Number(value)`，非法值返回 `NaN`，然后 `Number.isFinite(NaN)` 为 `false`，回退为 `0`。

**建议**：在 UI 中添加 `min`/`max` 约束（已有），并在输入框失去焦点时校验。`0` 作为回退值对于增量字段可能不合理（无标签增量为 0 意味着永远不增长），建议最小值为 1。

## 五、新增字段开发检查清单

为创意工坊添加新字段时，按以下清单逐项检查：

- [ ] **models/system.ts**：在 `ModeRuntimeProfile` 接口中添加字段（含 `?` 可选标记）
- [ ] **utils/modeRuntimeProfile.ts**：在 `构建官方模式运行时配置基础` 中添加默认值
- [ ] **utils/modeRuntimeProfile.ts**：在 `规范化模式运行时配置` 中添加规范化逻辑
- [ ] **components/features/Workshop/CreativeWorkshopModal.tsx**：在 `运行时配置分区列表` 中添加字段定义
- [ ] **类型渲染**：确认字段类型（text/textarea/list/bool/number 等）有对应的渲染分支
- [ ] **只读预览**：确认 `渲染只读运行时字段` 支持该字段类型
- [ ] **新建游戏写入**：如果需要在新建游戏时写入 `gameConfig`，在 `sessionLifecycleWorkflow.ts` 中添加写入逻辑
- [ ] **运行时消费**：确认消费方正确定义了读取路径
- [ ] **默认值一致性**：确保 `默认游戏设置`、`modeRuntimeProfile` 官方默认、`useGameState` 初始值三者一致
- [ ] **存读档**：如果字段存入 `gameConfig`，确认 `saveCoordinator` 和 `规范化游戏设置` 正确处理
- [ ] **UI 锁定**：如果字段需要创意工坊锁定机制，在 `GameSettings.tsx` 中添加对应的 prop 和 disabled 逻辑
- [ ] **文档更新**：更新本文档
