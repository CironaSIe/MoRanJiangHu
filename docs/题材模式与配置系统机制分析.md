# 题材模式与配置系统机制分析

## 一、题材模式系统（Topic Mode System）

### 1.1 架构概览

题材模式系统定义在 `data/workshopThemes/topicModeThemeData.ts` 中，通过 `题材模式配置表` 集中管理 8 种题材的全部配置。入口文件 `utils/topicModeProfiles.ts` 仅做 re-export。

```
topicModeThemeData.ts (数据源)
    ↓
topicModeProfiles.ts (统一出口)
    ↓
openingConfig.ts / modeRuntimeProfile.ts (消费方)
```

### 1.2 题材分组

8 种题材按 `group` 字段分为 7 个分组：

| 分组标识 | 题材 | 分组特征 |
|----------|------|----------|
| `wuxia` | 武侠 | 凡俗武道，内力招式 |
| `xianxia` | 仙侠 | 修真体系，灵根灵力 |
| `western_fantasy` | 西方奇幻 | 职业等级，魔法契约 |
| `urban_xianxia` | 灵气复苏 | 现代+修行，觉醒者 |
| `urban_xianxia` | 都市修仙 | 现代都市，隐秘修行圈 |
| `modern` | 现代都市 | 纯现实，职业人际 |
| `apocalypse` | 末日丧尸 | 生存压力，物资营地 |
| `infinite` | 无限流 | 主神空间，轮回小队 |

### 1.3 题材模式配置结构（题材模式配置）

每个题材配置包含以下关键字段：

| 字段 | 作用 |
|------|------|
| `value` / `label` / `shortLabel` | 题材标识、完整标签、短标签 |
| `hint` | 一句话描述题材特征 |
| `group` | 分组标识，影响运行时行为判断 |
| `worldDefaults` | 世界观生成默认值（世界名、版图大小、王朝设定、门派密度、天骄设定、额外约束） |
| `worldSizeLabel` / `worldSizeHint` | 版图尺度的 UI 文案 |
| `dynastyLabel` / `dynastyHint` | 格局设定的 UI 文案 |
| `densityLabel` / `densityOptions` / `densityPromptLabel` | 组织密度的 UI 选项 |
| `tianjiaoLabel` | 天骄/天赋设定的 UI 标签 |
| `auctionName` / `marketVerb` | 拍卖行名称和流通动词 |
| `currencyPrompt` / `currencyDisplayMode` / `currencyExchangePrompt` | 货币体系的三层描述 |
| `mapPrompt` | 地图生成提示词 |
| `skillNames` | 该题材可用的 8 种生活技能 |
| `backgroundSuggestions` | 5 种开局背景建议 |
| `talentSuggestions` | 5 种开局天赋建议 |
| `presetItemKeywords` | 预设物品关键词池 |
| `manualRealmPrompt` | 手动境界体系提示词 |
| `promptBoundary` | AI 生成边界限制（防止题材串味） |
| `promptLines` | 注入系统提示词的多行约束 |

### 1.4 题材规范化函数

- `规范化题材模式(mode)`：将任意输入规范化为有效的题材模式类型。别名映射（如"灵气修仙"→"灵气复苏"），无效值回退为"武侠"。
- `题材是否仙侠(mode)`：判断 group 是否为 `xianxia` 或 `urban_xianxia`。
- `题材是否现代(mode)`：判断 group 是否为 `urban_xianxia`、`modern` 或 `apocalypse`。

## 二、开局配置系统（OpeningConfig）

### 2.1 核心结构

`utils/openingConfig.ts` 定义了 `OpeningConfig` 的完整结构和规范化逻辑。

**默认开局配置** (`默认开局配置()`)：
- 继承"武侠"题材的默认开局配置
- 附加 `modeRuntimeProfile`（运行时配置）
- `允许生成性别`：默认 ['男', '女', '男娘', '扶她']
- `生成性别锁定`：默认 false

### 2.2 属性分配系统

6 种属性：`力量、敏捷、体质、根骨、悟性、福源`

| 函数 | 作用 |
|------|------|
| `创建默认属性分配()` | 返回默认值分配 |
| `创建平均属性分配(totalBudget)` | 将总点数平均分配到六维 |
| `创建随机属性分配(totalBudget, random)` | 随机分配剩余点数 |
| `计算属性总点数(attributes)` | 计算当前属性总和 |
| `规范化属性分配(value)` | 将任意输入规范化为合法属性结构 |

### 2.3 同人融合配置

```
同人融合: {
  enabled: boolean;           // 是否启用
  作品名: string;
  来源类型: '小说' | '动漫' | '游戏' | '影视';
  融合强度: '轻度映射' | '中度混编' | '显性同台';
  保留原著角色: boolean;
  启用角色替换: boolean;
  替换目标角色名: string;      // 替换为玩家名
  附加替换角色名列表: string[];
  附加角色替换规则列表: Array<{原名称, 替换为}>;
  启用附加小说: boolean;
  附加小说数据集ID: string;
}
```

### 2.4 角色替换规则

`获取同人角色替换规则列表(config, playerName)` 合并三类替换来源：
1. `替换目标角色名` → 替换为玩家名
2. `附加替换角色名列表` → 每个都替换为玩家名
3. `附加角色替换规则列表` → 自定义映射

所有规则通过 Map 去重，确保一一对应。

### 2.5 运行时快照（OpeningRuntimeSnapshot）

规范化后的快照包含：
- `openingStreaming`：是否流式开局
- `openingExtraRequirement` / `openingExtraPrompt`：额外约束/提示
- `activeModuleExtraRules`：模块额外规则
- `modeWorldbooks`：世界书条目
- `workshopSelection`：创意工坊选择
- `modeBackgrounds` / `modeTalents`：题材背景和天赋列表

## 三、游戏难度系统

### 3.1 五档难度

| 难度 | 起始属性点 | 天赋重Roll次数 | 判定修正 | 资源压力 | 失败代价 |
|------|-----------|---------------|---------|---------|---------|
| relaxed | 38 | 12 | +3 | 宽松 | 轻伤/可补救 |
| easy | 34 | 8 | +1 | 略高收益 | 大多能补救 |
| normal | 30 | 5 | 0 | 标准 | 伤势/关系损失 |
| hard | 26 | 3 | -1 | 高物价慢恢复 | 长期后遗症 |
| extreme | 22 | 1 | -3 | 最高压力 | 重伤/残废/主线断裂 |

### 3.2 题材化难度调整

`获取题材化难度设定(difficulty, mode)` 按题材调整描述：
- **无限流**：normal→"标准轮回"，资源压力改为"奖励点/支线剧情"
- **末日丧尸**：normal→"标准求生"，资源压力改为"补给/感染风险"
- **现代都市**：normal→"标准现实"，资源压力改为"资金/人情"

## 四、运行时配置系统（ModeRuntimeProfile）

### 4.1 完整结构

`utils/modeRuntimeProfile.ts` 中的 `ModeRuntimeProfile` 是题材模式的运行时展开形态，包含 12 个子系统：

```
ModeRuntimeProfile
├── identity        // 身份标识
├── economy         // 经济系统
├── time            // 时间系统
├── organization    // 组织系统
├── ability         // 能力系统
├── items           // 物品系统
├── map             // 地图系统
├── task            // 任务系统
├── npc             // NPC系统
├── image           // 生图系统
├── opening         // 开局系统
├── validation      // 校验系统
└── 叙事平静值配置   // 叙事节奏
```

### 4.2 各子系统详解

**identity（身份标识）**：
- `modeId` / `displayName` / `baseMode`：题材标识
- `isModern`：是否现代题材
- `usesCultivation`：是否修炼体系
- `isApocalypse` / `isSurvival`：是否末日/生存题材
- `isFandomIp`：是否同人IP

**economy（经济系统）**：
- `currencyDisplayMode`：货币显示模式（7 种）
- `currencyTiers`：三层货币（上/中/下层名称 + 汇率）
- `currencySystem`：完整货币体系（支持自定义覆盖）
- `marketName` / `marketVerb`：市场名称和流通动词
- `allowedItemTypes` / `bannedKeywords`：物品类型白名单/黑名单

**time（时间系统）**：
- `displayFormat`：显示格式（traditional/western/modern/apocalypse/infinite）
- `calendarName`：历法名称
- `narrativeStyle`：叙事风格提示词
- `dayPeriodNames`：8 个时段名称
- `allowedTimeTerms` / `bannedTimeTerms`：允许/禁止的时间词
- `progressionPrompt`：时间推进提示词

**organization（组织系统）**：
- `organizationName` / `memberName` / `contributionName`：组织/成员/贡献名称
- `rankNames`：6 级等级名称
- `organizationAliases` / `memberAliases`：组织/成员别名池

**ability（能力系统）**：
- `primaryAxis`：能力主轴描述
- `progressionNames`：5-6 级成长阶段名
- `attributePointRules`：属性点规则
- `skillPool`：8 种技能池
- `skillGrowthVerb`：技能成长动词
- `combatResolution`：战斗结算规则

**items（物品系统）**：
- `initialItemPool` / `rewardItemPool`：初始/奖励物品池
- `bannedItemKeywords` / `exclusiveItemTypes`：禁用词/独占类型
- `resourceToggles`：7 种资源开关（food/water/ammo/medicine/fuel/batteries/spiritStones）
- `activeResources`：活跃资源计数器列表

**map（地图系统）**：
- `layerNames`：六层地图（寰宇→大地点→中地点→小地点→区地点→子地点）
- `locationTypes`：地点类型列表
- `poiTypes`：POI 类型列表
- `bannedLocationKeywords`：禁用地点关键词
- `mapPrompt`：地图生成提示词

**task（任务系统）**：
- `mainQuestStyle`：主线风格
- `sideQuestDedupeKeys`：支线去重键
- `rewardDistributor`：奖励发放者
- `rewardVisualizationTemplate`：奖励可视化模板

**npc（NPC系统）**：
- `defaultIdentityPool`：默认身份池
- `relationTemplates`：关系模板
- `requiredMainCharacterFields`：主要角色必填字段（10 个）
- `sexualityFallback` / `sensitivityFallback`：性癖/敏感点回退规则
- `autoImageStyle`：自动生图风格
- `genderRatio`：性别比例（字符串 "1:3" 或对象）

**image（生图系统）**：
- `characterClothingEra`：服饰时代
- `sceneMaterials`：场景材质
- `itemRealismPrompt`：物品写实提示词
- `negativePrompt`：负面提示词
- `visualStyle`：视觉风格

**opening（开局系统）**：
- `defaultBackgrounds` / `defaultTalents`：默认背景/天赋
- `companionTemplate`：伙伴模板
- `cutInTemplates`：切入模板（5 种）
- `initialQuestTemplates`：初始任务模板
- `allowedGeneratedGenders` / `lockGeneratedGenders`：生成性别控制

**validation（校验系统）**：
- `bannedWords`：禁用词
- `conflictChecks`：冲突检测项
- `migrationCleanupRules`：迁移清理规则

### 4.3 货币系统模板

`构建货币系统模板(templateId, profile)` 支持 9 种预设：

| 模板 ID | 说明 | 结构 |
|---------|------|------|
| `topic-default` | 题材默认 | 三层动态 |
| `single` | 单一货币 | 单币种 |
| `modern-yuan` | 人民币 | 单币种 ¥ |
| `credit` | 信用点 | 单币种 |
| `wuxia` | 武侠金银铜 | 三层 1:1000:100000 |
| `xianxia` | 修仙灵石 | 四层 1:1000:100000:100000000 |
| `fantasy` | 西幻金银铜 | 三层 1:100:10000 |
| `apocalypse` | 末日物资券 | 三层 1:10:1000 |
| `infinite` | 无限流奖励点 | 三层 1:1000:100000 |

### 4.4 规范化流程

`规范化模式运行时配置(raw, fallbackMode)` 是核心入口：
1. 先通过 `构建官方模式运行时配置基础(baseMode)` 生成该题材的完整默认值
2. 逐字段用 raw 值覆盖官方默认值
3. 每个子系统都有独立的文本/布尔/正整数/列表规范化
4. 确保输出始终是完整合法的 ModeRuntimeProfile

## 五、API 配置系统（apiConfig.ts）

### 5.1 供应商类型

```typescript
type 接口供应商类型 = 'gemini' | 'claude' | 'openai' | 'deepseek' | 'zhipu' | 'mimo_api' | 'mimo_token_plan' | 'openai_compatible'
```

### 5.2 功能模型占位配置

`默认功能模型占位` 定义了所有可独立配置模型/API 的功能开关：

- **主剧情**：模型、非流式输出
- **DeepSeek 稳定模型救场**：开关 + 模型/API地址/密钥
- **GLM 稳定模型救场**：同上
- **剧情回忆**：独立模型开关 + 静默确认 + 条数 + 触发回合
- **记忆总结/精炼**：独立模型 + 渠道ID + API
- **世界演变**：独立模型开关 + 功能启用
- **变量计算**：独立模型 + 功能启用
- **规划分析/女主规划/剧情规划**：独立模型开关
- **文章优化**：独立模型 + 提示词
- **小说拆分**：独立模型 + 批量参数 + 注入开关
- **地图生成**：独立模型 + 功能启用
- **地图自动更新**：独立模型开关
- **文生图**：后端类型 + 模型 + 接口路径 + 响应格式
- **场景/NPC/NSFW 生图**：各自独立接口配置
- **词组转化器**：独立模型 + 启用开关

### 5.3 图像生成管线

**画师串预设** (`默认画师串预设列表`)：
- 3 个默认预设，覆盖 all/scene 范围
- 包含 GPT Image 2 通用/场景的正面/负面提示词

**词组转化器预设** (19 个)：
- 覆盖 5 种后端：NAI、GeminiBanana、Grok、ComfyUI、GPT Image 2
- 每个后端有 npc/scene/scene_judge 三类预设
- 包含：主提示词、锚定模式提示词、无锚点回退提示词、输出格式提示词

**模型词组转化器预设** (5 个)：
- 按模型类型分 bundle
- 包含：模型专属提示词、锚定模式提示词
- 词组序列化策略：`nai_character_segments` / `gemini_structured` / `grok_structured` / `flat`

### 5.4 词组序列化策略

| 策略 | 适用后端 | 输出结构 |
|------|---------|---------|
| `nai_character_segments` | NovelAI | `<基础>` + `[序号]角色\|tags` |
| `gemini_structured` | Gemini | 描述式英文短语 |
| `grok_structured` | Grok | 电影感描述式提示词 |
| `flat` | ComfyUI/GPT Image 2 | 逗号分隔的英文 tags |

**仙侠默认映射**（43 级）：

### 6.3 境界母板优先级

`读取境界母板优先级` 的查找顺序：
1. 独立境界体系提示词中的【境界映射母板】
2. world_prompt 中的【境界映射母板】
3. 默认武侠/仙侠母板

### 6.4 同人模式补丁机制

当 `同人融合.enabled === true` 时，系统生成 12 个补丁文本：

| 补丁 | 注入位置 | 作用 |
|------|---------|------|
| 世界观创建补丁 | 世界观生成阶段 | 强制先贴原著再偏转 |
| 开局任务补丁 | 第一幕生成前 | 核对原著时期/地点/人物 |
| 开局COT补丁 | 开局思考链 | 增加"当前章节内容对照"步骤 |
| 主剧情COT补丁 | 每回合主思考链 | Step0 复核原著基线 |
| 剧情规划补丁 | 规划分析 | 优先承接原著矛盾链 |
| 女主规划补丁 | 女主线规划 | 校验原著女性角色边界 |
| 女主思考补丁 | 女主思考 | 回答是否贴原著 |
| 世界演变补丁 | 后台世界推进 | 遵守原著势力逻辑 |
| 境界母板补丁 | 境界体系 | 独立境界映射 |
| 变量校准补丁 | 变量生成 | 不自动修正回默认武侠术语 |

### 6.5 融合强度策略

| 强度 | 效果 |
|------|------|
| 轻度映射 | 只借设定气质与世界母题 |
| 中度混编 | 允许部分势力/设定进入原创世界 |
| 显性同台 | 允许原著角色/势力直接存在 |

### 6.6 境界体系区块替换

`应用境界体系区块替换(sourceText, bundle)` 运行时替换系统提示词中的默认武侠境界文本：
- 替换 8 类默认提示词为当前存档专属版本
- 仙侠模式额外替换 3 处关键文案（体系标题、修炼主轴、硬边界标题）

## 七、配置流转总览

```
用户选择题材模式
    ↓
题材模式配置表 → 题材模式配置
    ↓
    ├─→ 合并题材世界默认值 → WorldGenConfig
    ├─→ 构建官方模式运行时配置 → ModeRuntimeProfile
```
