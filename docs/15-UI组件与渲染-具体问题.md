# UI 组件与渲染系统——具体代码级问题

## 1. InputArea.tsx — 输入框

### 问题 1.1：StrictMode 下 interval 可能泄漏
**文件**: `components/features/Chat/InputArea.tsx:231-244`
```typescript
useEffect(() => {
    if (loading || postStoryQueueRunning) {
        if (!mainStoryStartRef.current) { mainStoryStartRef.current = Date.now(); }
        const timer = setInterval(() => {
            setMainStoryElapsed(Date.now() - mainStoryStartRef.current);
        }, 200);
        return () => clearInterval(timer);
    } else {
        setMainStoryElapsed(0);
        mainStoryStartRef.current = 0;
    }
}, [loading, postStoryQueueRunning]);
```
`loading` 和 `postStoryQueueRunning` 独立变化，React 18 StrictMode 下 effect 双执行可能导致一个 interval 未被清理。

### 问题 1.2：`externalDraft` effect 依赖不完整
**文件**: `components/features/Chat/InputArea.tsx:282-290`
```typescript
useEffect(() => {
    if (!externalDraft?.text) return;
    setContent((current) => {
        const draft = externalDraft.text.trim();  // ← 访问 text
        // ...
    });
}, [externalDraft?.token]);  // ← 但依赖只有 token
```
`token` 不变但 `text` 变化时 effect 不重新执行。

### 问题 1.3：`onStageFailureDecision` 重复定义
**文件**: `components/features/Chat/InputArea.tsx:385-403` 和 `415-433`
两处 `onStageFailureDecision` 回调逻辑完全相同但分别定义。修改时易遗漏。

### 问题 1.4：try 内 return 导致 lastSentContent 不更新
**文件**: `components/features/Chat/InputArea.tsx:373-491`
```typescript
try {
    if (result?.cancelled) {
        if (result.needRerollConfirm) { ...; return; }  // ← try 内 return
        ...; return;  // ← finally 执行 setIsPreparing(false)
    }
} finally { setIsPreparing(false); }
```
取消后 `setLastSentContent(content)` 和 `setContent('')` 不执行。用户点"停止"时恢复的是上一次成功发送的内容。

### 问题 1.5：无用 effect
**文件**: `components/features/Chat/InputArea.tsx:771, 801`
useEffect 仅做状态判断（空 if 分支），无实际副作用，属于无用 effect。

## 2. MessageRenderers.tsx — 消息渲染

### 问题 2.1：`dangerouslySetInnerHTML` 无运行时安全验证 🔴
**文件**: `components/features/Chat/MessageRenderers.tsx:1239-1244`
```tsx
if (htmlRenderMode === 'purify') {
    return (
        <div dangerouslySetInnerHTML={{ __html: htmlContent }} />
    );
}
```
`htmlContent` prop 注释说"已 DOMPurify 清洗"，但没有运行时验证。上游忘记清洗时 XSS 直接发生。

### 问题 2.2：`渲染含档案引用文本` 未 memo 化
**文件**: `components/features/Chat/MessageRenderers.tsx:581`
```typescript
const displayText = useMemo(() => 格式化旁白断行(text).replace(...), [text]);
// 但渲染含档案引用文本(displayText, inventoryItems, socialList, ...) 返回值未 memo
```
长文本 + 大量物品列表时，每次渲染都重新扫描 `《》` 标记并做物品匹配。

### 问题 2.3：`CharacterRenderer` 未 memo 化
**文件**: `components/features/Chat/MessageRenderers.tsx:778`
```typescript
const displayText = (text || '').replace(/\s*\n+\s*/g, ' ').replace(/[ \t]{2,}/g, ' ').trim();
```
每次渲染执行两次正则替换。20 个角色消息 = 40 次不必要运算。

## 3. ChatList.tsx — 聊天列表

### 问题 3.1：ref 对象长期持有旧 DOM 引用
**文件**: `components/features/Chat/ChatList.tsx:111-112`
```typescript
const 回合容器Refs = React.useRef<Record<number, HTMLDivElement | null>>({});
const 消息容器Refs = React.useRef<Record<number, HTMLDivElement | null>>({});
```
`sliceIndex` 从 0 变为 10 时，索引 0-9 的 DOM 已卸载但 ref 仍持有引用。长期游戏（数百回合）内存持续增长。

### 问题 3.2：滚动位置在异步内容加载前获取
**文件**: `components/features/Chat/ChatList.tsx:353-354`
```typescript
raf1 = window.requestAnimationFrame(() => {
    raf2 = window.requestAnimationFrame(() => {
        const nextTop = Math.max(0, targetEl.offsetTop - 12);
        container.scrollTop = nextTop;
    });
});
```
目标元素包含图片时，`offsetTop` 在图片加载前获取，滚动位置偏小。图片加载后高度增加但滚动位置不更新。

## 4. SandboxedCard.tsx — 沙箱卡片

### 问题 4.1：`postMessage` 使用 `'*'` targetOrigin
**文件**: `components/features/Chat/SandboxedCard.tsx:105, 125, 157`
```typescript
iframe.contentWindow?.postMessage({ type: 'tavern_sandbox_response', ... }, '*');
```
任何窗口都能接收到消息。当前只传递主题变量（低风险），但未来扩展为传递敏感数据时成为信息泄露通道。

### 问题 4.2：`measureHeight` try-catch 吞掉所有错误
**文件**: `components/features/Chat/SandboxedCard.tsx:170-182`
```typescript
try {
    const doc = iframe.contentDocument;
    if (doc?.body) { const h = doc.body.scrollHeight; ... }
} catch { /* 跨域限制下不可访问 */ }
```
`srcDoc` 是同源内容，`contentDocument` 实际可访问。catch 吞掉所有错误，iframe 高度卡在初始值 60px 无提示。

### 问题 4.3：`iframeHeight` 依赖导致 interval 频繁重建
**文件**: `components/features/Chat/SandboxedCard.tsx:165-193`
```typescript
useEffect(() => {
    const measureHeight = () => {
        if (h > 0 && h !== iframeHeight) { setIframeHeight(Math.min(h, maxHeight || 2000)); }
    };
    const timer = setTimeout(measureHeight, 200);
    const interval = setInterval(measureHeight, 2000);
    return () => { clearTimeout(timer); clearInterval(interval); };
}, [isReady, maxHeight, iframeHeight]);  // ← iframeHeight 在依赖中
```
每次高度变化都重建 interval 和 timer。iframe 内容高度频繁变化时 interval 被反复重置，measureHeight 可能永远不执行。

## 5. SocialModal.tsx — NPC 社交面板

### 问题 5.1：大量 console.log 调试代码残留
**文件**: `components/features/Social/SocialModal.tsx:186-229, 319-343`
```typescript
useEffect(() => {
    if (!currentNPC) return;
    console.group(`[私密图调试] NPC: ${npcName}`);
    console.log('条件状态:', { ... });
    console.log('图片档案:', archive ? '存在' : '不存在');
    console.groupEnd();
}, [currentNPC?.id, nsfwEnabled, femboyNsfwEnabled]);
```
每次 NPC 变化都打印大量对象。低端设备卡顿 + 敏感数据泄露到控制台。

### 问题 5.2：`indexOf` 在 sort 回调中导致 O(n²)
**文件**: `components/features/Social/SocialModal.tsx:84-90`
```typescript
const sortedSocialList = React.useMemo(() => (
    [...socialList].sort((a, b) => {
        const weightDiff = 计算社交排序权重(a) - 计算社交排序权重(b);
        if (weightDiff !== 0) return weightDiff;
        return socialList.indexOf(a) - socialList.indexOf(b);  // ← O(n) per comparison
    })
), [socialList]);
```
sort 本身 O(n log n)，每次比较 indexOf O(n)，总体 O(n² log n)。100+ NPC 时排序耗时显著。

### 问题 5.3：`当前记忆展示` useMemo 依赖整个对象
**文件**: `components/features/Social/SocialModal.tsx:168-171`
```typescript
const 当前记忆展示 = React.useMemo(
    () => currentNPC ? 构建NPC记忆展示结果(currentNPC.总结记忆, currentNPC.记忆) : ...,
    [currentNPC]  // ← 整个对象引用
);
```
`currentNPC` 引用变化但记忆未变时仍重新计算。

## 6. GameSettings.tsx — 游戏设置

### 问题 6.1：`实时应用更新` 竞态条件 🔴
**文件**: `components/features/Settings/GameSettings.tsx:145-149`
```typescript
const 实时应用更新 = (patch) => {
    const next = { ...form, ...patch };
    setForm(next);    // ← 异步
    onSave(next);     // ← 同步
};
```
快速连续切换多个 toggle 时，多次 `onSave` 基于同一个 `form` 快照的增量更新，可能产生竞态。

### 问题 6.2：`settings` prop 变化覆盖本地编辑
**文件**: `components/features/Settings/GameSettings.tsx:113-116`
```typescript
useEffect(() => {
    setForm(settings);
    setWordCountDraft(String(settings.字数要求 ?? ''));
}, [settings]);
```
父组件因其他原因重渲染传入新 `settings` 引用时，覆盖用户正在编辑但未保存的 `form`。

### 问题 6.3：字数要求 onBlur 才提交
**文件**: `components/features/Settings/GameSettings.tsx:508-519`
用户输入数字后直接关闭弹窗（不 blur 输入框），修改丢失。

### 问题 6.4：`额外提示词` onBlur 闭包捕获过期 form
**文件**: `components/features/Settings/GameSettings.tsx:931-941`
```typescript
onBlur={(e) => {
    const next = { ...form, 额外提示词: e.currentTarget.value };  // ← form 可能是旧快照
    setForm(next);
    onSave(next);
}};
```
`setForm` 异步，`onBlur` 触发时 `form` 可能不包含用户刚输入的最新字符。
